# uiable-mcp

MCP server for the [uiable](../../README.md) shadcn registry. It exposes tools so AI
assistants (Claude Code, Cursor, VS Code, …) can **discover**, **install**, and
**generate** UI from uiable blocks via natural language.

This is the "Layer 2" custom server. Layer 1 (the standard 7 registry tools) comes for
free from `npx shadcn@latest mcp` once a consumer has `@uiable` configured — see the
repo's registry docs.

## Tools

| Tool | Purpose |
| --- | --- |
| `list_blocks` | List page-level blocks (hero, pricing, feature, footer…), filter by category. |
| `list_components` | List primitive components (button, input, card…), filter by category. |
| `search_blocks` | Ranked keyword/intent search across all items, with previews + install commands. |
| `get_block` | Full details for one item: metadata + source file contents + add command. |
| `get_add_command` | `npx shadcn add` command(s) for one or more items (validates names). |
| `create_ui` | Compose a page from blocks for an intent: ordered plan + `page.tsx` scaffold + add commands. |

## Build

```bash
# from repo root, deps already resolve from root node_modules
node node_modules/typescript/bin/tsc -p packages/uiable-mcp/tsconfig.json
# smoke test
cd packages/uiable-mcp && node smoke-test.mjs
```

## Catalog source

The server reads `registry-index.json` (generated by `npm run registry:catalog`).
Resolution order:

1. `UIABLE_CATALOG` — explicit URL or file path to `registry-index.json`.
2. `UIABLE_BASE_URL` — hosted base, e.g. `https://uiable.com` → fetches `/registry-index.json` and `/r/<name>.json`.
3. **Local fallback** (default, pre-publish) — the repo's `public/registry-index.json` and `public/r/`.

So today it works fully offline against the local build. When uiable is hosted, set
`UIABLE_BASE_URL` and it serves the live catalog with no code change.

## Wire into a client

**Claude Code** (`.mcp.json` / `claude mcp add`):

```json
{
  "mcpServers": {
    "uiable": { "command": "npx", "args": ["uiable-mcp"] }
  }
}
```

**Cursor** (`.cursor/mcp.json`) / **VS Code** (`.vscode/mcp.json`): same shape.

For local dev before publishing to npm, point at the built entry directly:

```json
{
  "mcpServers": {
    "uiable": { "command": "node", "args": ["packages/uiable-mcp/dist/index.js"] }
  }
}
```
